# 「澱みのスープ」 ゲームレビュー（プレイ感・バグ・UI）

> **レビュー対象**: 通常ルート全体（intro → unlocked → news_found → location_spec → chat_sent → clear/bad_end）＋ extraモード（/extra → 白菊HD調査 → true_end）  
> **レビュー日**: 2026-06-28

---

## 1. プレイ感（ゲームとしての体験）

### 🌟 特に優れている点

- **デバイス切り替えの没入感**  
  「自分のスマホ」と「拾ったスマホ」を行き来するコアシステムは独自性が高く、ARG（代替現実ゲーム）としての没入感が極めて高い。どちらのデバイスで何をすべきかが徐々に明確になる進行設計も秀逸。

- **パスコード謎解きの達成感**  
  SNSの投稿（誕生日）とケースの刻印（オープン日）から `19940826` を推理する導線が自然。「情報を組み合わせて自分で解く」という純粋な謎解きの醍醐味がある。

- **ブラウザのホラー演出**  
  「秘伝の骨」クリック → 店名が「ジンニクじんこつ」に文字化けするギミックは絶妙なタイミングで発動し、ゲームの核心を視覚的に伝える。`pulseHint` アニメーションによるアフォーダンスも適切（ニュース発見後に限定して有効化される点も丁寧な設計）。

- **バッドエンド演出の質**  
  店主の「…おや？」「あなたは……太郎さんではありませんね？」「みいつけた」という段階的メッセージ → 背景が赤く染まる演出 → ジャンプスケア（赤い目のシルエット）→ グリッチカバーの順序が練られており、恐怖を段階的に高めるリズムが良い。

- **extraモードの設計**  
  グッドエンドで終わらせず「黒幕（白菊ホールディングス）」という次の謎へ誘導する構成は、プレイヤーへの正直な報酬となっている。太郎とのチャットというインタラクションへの接続も自然。

### 🔧 プレイ体験上の課題

#### 1. 鏡撮影フローが説明不足で詰まりやすい
- バッドエンド後のリトライ時、「自分のスマホのカメラで鏡を撮影する」というアクションが分かりにくい。
- 友人チャットのヒント（idx 15〜17）で導線を引いているが、**自分のスマホのホーム画面に「カメラ」アイコンがあることに気づかないまま詰まる**プレイヤーが出やすい。
- **推奨改善**: 友人の「カメラで鏡撮影してみろよ！」というメッセージが届いた後、自分のスマホのカメラアイコンに `!` バッジを表示する。

#### 2. extraモードでの「拾ったスマホを見る」ボタン非表示
- extraモード（`/extra`）では `flipBtn.style.display = 'none'` により切り替えボタンが非表示になるが、その理由の説明が一切ない。
- バッテリー切れで見られなくなった、という文脈がある程度伝わるが、明示があるとより親切。

#### 3. 白菊HDサイトの「隠し文字パズル」の難易度と手がかり不足
- `shirakiku.html` に7文字のクリックパズル（`c1`〜`c7`）が仕込まれており、全部クリックすると「極秘出荷管理コード一覧」が解放される。
- しかし **プレイヤーが「文字をクリックできる」と気づかない可能性が高い**。フォントが通常テキストと同じで、hover/cursor のフィードバックがない。
- この一覧から社員ログインID（`RM-9942`）とパスワード（`0900`）が得られる連鎖が設計されているが、発見されずに素通りされるリスクが高い。

## 2. バグ・実装上の不具合

### 🔴 要確認のバグ

#### [BUG-01] シェアリンクのURLがlocalhost固定（`share-true-x` は未設定）
`setupShareLinks()`（`os-core.js`）は `share-clear-x` と `share-bad-x` を正しいURLで上書きするが、**`share-true-x`（トゥルーエンド）は設定されていない**。HTMLに `http://localhost:4321/` がそのまま残る。

**修正案**: `setupShareLinks()` に `share-true-x` の設定を追加。
```js
const trueLink = document.getElementById('share-true-x');
const trueText = encodeURIComponent("白菊ホールディングスの巨大な闇を暴き...");
if (trueLink) {
  trueLink.href = `https://x.com/intent/tweet?text=${trueText}&url=${encodeURIComponent(gameUrl)}`;
}
```

#### [BUG-02] グッドエンドのエピローグ本文の住所が誤り（`3-19-4` → `3-19-3`）
`index.html`（templates/index.html）315行目のエピローグ本文:
```
警察の介入により、台東区音無町3-19-4の地下室から…
```
グッドエンドは正しい住所（`3-19-3`）を送って救出されるルートなので本文も `3-19-3` であるべき。`3-19-4` はケンが罠にはまったバッドエンド側の住所。

#### [BUG-03] `inspectPlayerMirrorPhoto()` が `trueAddressFound` フラグを立てない
`os-core.js` の `inspectPlayerMirrorPhoto()` は調査テキストを表示するだけで `setFlag('trueAddressFound', true)` を呼ばない。正規ルートは「拾ったスマホのカメラ → 拾ったスマホのギャラリーで調査」であり、自分のスマホのギャラリーは見た目確認用として意図的な設計とも取れるが、コメントでの明示を推奨。

#### ~~[BUG-04] バッドエンド開始インデックスのズレ（`29` → `27`）~~ ✅修正済み
`story_review.md` で `playerChatIndex = 29` が指摘されていたが、コードを確認したところ `27` に正しく修正済み。

#### [BUG-05] `renderPlayerChat()` でリロード後 `clear` ステップのメッセージが不完全 ✅本レビューにて修正済み
`stepIndexMap['clear'] = 19` の設定では、リロード後に idx 0〜18 しか描画されず、グッドエンドの充電切れメッセージ（idx 19〜21）が欠落していた。`stepIndexMap['clear'] = 22` に変更し、`renderPlayerChat()` に friend スレッドフィルタ（`thread === 'friend'`）を追加して修正。

#### [BUG-06] `gallery.html` のCSS構文エラー（孤立した `}`）
`gallery.html` 111行目に余分な `}` がある。描画には影響しないが構文エラー。

#### [BUG-07] `gallery.html` の `--muted` 変数が二重定義
16行目に `--muted: #6a6a80;` が2回連続している。機能的影響はないが冗長コード。

### 🟡 軽微な懸念点

#### [WARN-01] `shirakiku.html` → `TRIGGER_TRUE_END` の多重postMessage送信
`triggerTrueEnd()` が `window.top` / `window.parent` / `window.parent.parent` の3経路で送信しており（2段階iframeネスト対策）、多重発火しないか確認を推奨。

## 3. UI・UXの問題

### 📱 モバイル全般

- `browser.html`：`@media (max-width: 768px) { #content { padding-bottom: 60px; } }` 対応済み ✅
- `gallery.html`：`#gallery-body { padding-bottom: 65px; }` 対応済み ✅
- `chat.html`：`#message-list { padding-bottom: 70px; }` 対応済み ✅
- **`sns.html` の下部**: 長いタイムラインの末尾がホームバーに隠れる可能性あり。要確認。

### 🎨 デザイン上の提案

- **カメラのビューファインダーが機能的すぎる**: 拾ったスマホのカメラ画面では、ケースの刻印テキスト（`台東区音無町3-19-3`）がシャッター前から見えてしまっている。撮影後に初めてズームして読める演出にした方が発見の喜びが増す。
- **カメラアイコンへのバッジ誘導**: 鏡撮影タイミング（友人チャット idx 17 受信後）に自分のスマホのカメラアイコンへバッジを付けると詰まり防止になる。

## 4. extraモード固有のレビュー

### 🌟 良い点

- **白菊ホールディングスのサイトのクオリティ**が高く、プロフェッショナルな企業サイトの体裁を完璧に再現。「地下施設を含む不動産」「骨髄エキス」という伏線の埋め込み方も巧み。
- **多段階の謎解き**（「隠し文字 → コード一覧解放 → 社員ログイン → ダークシステム → 告発ボタン」）が丁寧に設計されている。
- **extra-intro-overlay のナレーション**が自然な続編への誘導になっている。

### 🔧 extraモード固有の課題

#### 1. 隠し文字パズルの視認性が低すぎる（UX上の主要課題）
`.hidden-char` のスタイルが定義されておらず、通常テキストと見た目が同一。意図的な高難度設定としても、全く気づかれないリスクが高い。  
**推奨**: hover時に `cursor: pointer` + 微妙な色変化を最低限追加。

#### 2. extraモードでの次の行動への誘導がない
idx 26（太郎「調べてみます！」送信後）から白菊ブラウザを自力で開く必要があるが、その導線が不明瞭。  
**推奨**: idx 26 の sent 後にバッジや「ブラウザ通知」のトーストを表示。

#### 3. トゥルーエンドの `confirm()` ダイアログが没入感を下げる
`triggerTrueEnd()` 内の `confirm()` はブラウザネイティブのダイアログで演出のテンションを下げる。カスタムダイアログへの置き換えを推奨。

## 5. ストーリー・テキストの評価

### 🌟 特に優れている点

- **「ジンニクじんこつ（人肉人骨）」という命名の残酷さ**: 「〇ンニク〇んこつ」という伏字が解読されたときの衝撃が計算され尽くしている。
- **SNS投稿の伏線**: 「最近体がだるいが、あの店のスープがやたら飲みたくなる」が後から全て恐ろしい意味を持つ構造。
- **バッドエンドの「第四の壁破壊」**: 「この端末をご覧になっているのは…どなたです？」というプレイヤー自身への語りかけは恐怖のピーク。
- **「お待ちしておりますが」の未完文**: 意図的ホラー演出として維持を強く推奨。

### 🔧 気になる点

- **グッドエンドのエピローグ住所ミス**（[BUG-02] 参照）: `3-19-4` → `3-19-3` に修正が必要。
