# 【開発依頼・詳細要件定義】仮想OS型ARG『澱みのスープ（仮）』

ニンニクげんこつラーメン　というラーメンがあるが、それを一部伏せ字にすることで「人肉人骨ラーメン」を想起させることをコンセプトにしたい。

## 1. プロジェクト概要

本プロジェクトは、1つのドメイン内でPCおよびスマートフォンの「仮想OS（デスクトップ／モバイル）」を擬似的に再現し、プレイヤーに端末をハッキング・探索させる体験を提供するARG（代替現実ゲーム）である。
すべてフロントエンド（HTML/CSS/JS）のみの静的ファイルで構成し、**Cloudflare Pages**へのデプロイを前提とする。

---

## 2. アーキテクチャとファイル構造（推奨）

親ウインドウ（OSシェル）と子画面（アプリ）は、`iframe`のメッセージング（`postMessage`）または`localStorage`を介して状態（フラグ）を共有する。ゲームの雰囲気を壊さないよう、各htmlファイルへの直接アクセスはメイン画面へのリダイレクト等の処理を適切に行うこと。

### ディレクトリ構成案

* **dist/**
* **index.html** ： メインOSシェル（デバイス判定、ロック画面、通知、波演出）
* **_redirects** ： Cloudflare Pages用リダイレクト設定
* **assets/**
* **css/style.css** ： 共通・OS用スタイル
* **js/os-core.js** ： フラグ管理、ウィンドウ管理、音声制御
* **audio/notification.mp3** ： 通知音


* **apps/** ： 各種アプリ（iframe用埋め込み画面）
* **sns.html** ： X風SNS（配信者の過去ログ、ニュース引用）
* **browser.html** ： ダミーアドレスバー付きブラウザ（ラーメン店公式、Wiki等）
* **chat.html** ： LIME風チャット（相方との会話、自動メッセージ受信）
* **gallery.html** ： 証拠フォルダ（画像ビューア）

## 3. 画面仕様および実装詳細

### 3-1. デバイス判定とレスポンス

* **PCモード（画面幅 769px 以上）:**
* マルチウィンドウシステム。各アプリは `div.window`（内部に `iframe`）として動的に生成・表示。
* ドラッグによる移動、 `z-index` による前面背面制御、タスクバーでの最小化/復元を実装。


* **スマホモード（画面幅 768px 以下） ※メイン基盤:**
* 全画面シングルタスク。アプリ起動時は画面全体を覆う。
* 最下部に「ホームバー（ `div.home-bar` ）」を常駐させ、タップ時にアクティブなアプリを非表示（または下部スライドアウト）にしてホーム画面に戻す。



### 3-2. 背景演出（不気味さの視覚化）

* OS全体の背景（壁紙レイヤーの上）に、複数のSVG path（波形）を重ねて配置。
* CSS animation または JS を用い、それぞれの波を「異なる振幅」「異なる周期（スピード）」で不規則にうねらせることで、プレイヤーに微細な不快感・違和感（精神的澱み）を与える演出を施すこと。

## 4. 各アプリケーション要件 ＆ フラグ管理

ゲームの進行状況は `gameState` オブジェクトで管理し、 `localStorage` にてオートセーブを行う。また、デバッグおよびユーザー進捗保存用として、 `gameState` をJSONとしてエクスポート/インポートできる隠しメニュー（またはOS設定メニュー）を実装すること。

### 管理するゲーム状態（フラグ）の例

* isLocked: true （ロック画面フラグ）
* currentStep: 'intro' （進捗管理：intro, unlocked, news_found, location_spec, chat_sent, clear）
* notifiedItems: [] （既に出した通知のID）

### ① ロック画面（index.html 内レイヤー）

* **初期状態:** ゲーム起動時はこの画面が最前面を覆う。
* **ギミック:** 画面上部に「限定枠残り1席。今すぐ予約できます」というダミー通知（トースト）を表示させておく。
* **解除条件:** 「4桁のパスコード」を入力。
* 正解コードのロジック：配信者の誕生日 ＋ ラーメン店のオープン日 ＝ 4桁の数字（※数値はシナリオに応じて決定）。


* **音声制限解除:** この画面での最初のタップ（数字入力やスワイプ）をもって、ブラウザの音声再生制限を解除（AudioContextの有効化等）し、以降の通知音がバックグラウンドでも鳴るように制御すること。

### ② X風 SNSアプリ（apps/sns.html）

* **タイムライン構成:** 配信者「＠タベアルキ太郎」の過去ログ。
* **配置すべき重要コンテンツ:**
1. **プロフィール欄:** 誕生日の記載（ロック解除のヒント①）。
2. **周年記念ポスト:** 毎年恒例の「〇周年記念！」の写真。背景の店主の妻が年々窶（やつ）れ、最後には姿を消している。代わりに店主が形見（指輪）をつけている不穏なビジュアル。ここに「オープン日」のヒント（ロック解除のヒント②）。
3. **引用リポスト（ニュース）:** 【火葬場からの遺体盗難ニュース】のリンク（ダミー）を引用し、「この事件、店の近くだ…最近変な匂いがする」という不安の吐露。タップすると【ブラウザアプリ】でニュース詳細が開く。



### ③ ブラウザアプリ（apps/browser.html）

* **構成:** 上部にダミーのURL入力欄（編集不可、読み込み先に応じてJSで書き換え）。
* **表示コンテンツ:**
* SNSのリンクから遷移する「ニュース詳細ページ」。
* ラーメン店の「公式ホームページ」（裏マニュアルの伏字「〇ンニク〇んこつ」が、フラグ進行によって「人肉人骨」へと書き換わる、あるいは特定操作で文字化けが直るギミックを実装）。



### ④ 証拠フォルダ ＆ 画像ビューア（apps/gallery.html）

* スマホ内の「写真」アプリ。グリッド状のサムネイル。
* **最重要写真:** 配信者が最後に撮影した「店内の厨房写真」。
* 窓の外の反転した看板、壁のカレンダーの不自然な印など、場所（店舗の住所）を特定するための視覚的ヒントを画像内に明示的に作り込むこと。



### ⑤ LIME風 チャットアプリ（apps/chat.html）

* **構成:** 配信者の相方（チャンネルの相棒など）とのトーク画面。
* **クライマックスの挙動:**
* プレイヤーが【画像ビューア】で監禁場所（住所）を特定した後、このチャット画面を開くと「メッセージ入力欄（または選択肢）」が有効化される。
* 特定した住所と証拠写真を相方に「送信」するアクションを実行できる。
* 送信後、相方から「マジかよ、今すぐ警察と現地に向かう！」という緊迫したテキストが既読と同時に返信される。
* この返信の直後、画面全体を暗転（フェードアウト）させ、数日後の後日談（無事救出され、事件を告発する配信のアーカイブ画面など）を表示してゲームクリア（エピローグ）となる。
* *バッドエンド導線:* 住所を間違えたまま送信する、あるいは特定の禁忌に触れすぎると、店主からメッセージ（「みいつけた」等）が届き強制終了する分岐を設ける。

## 5. 開発における重要・よしなに実装してほしいポイント

1. **アセットのプリロード:** ロック画面の裏（ゲーム起動直後のローディング中）で、壁紙画像、アプリ用アイコン、通知音声（ `notification.mp3` ）を事前にキャッシュし、探索中に画面がガタ出たり、音が遅れて鳴ったりするのを防いでください。
2. **iframe間の通信最適化:** `window.parent.postMessage` を用いて、子画面での行動（例：ニュースを読んだ、写真を拡大した）が確実に親画面の `gameState` に反映され、適切なタイミングで「上部からの通知トースト（音付き）」が発生するよう、クリーンなイベントリスナーを構築してください。
