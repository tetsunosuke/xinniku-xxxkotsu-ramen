## 開発指示書: 仮想OS型ARGホラーゲーム「澱みのスープ」

AstroとVanilla JavaScript（HTML/CSS含む）を用いて、スマートフォンの仮想OSを模したブラウザベースのアドベンチャーホラーゲームを作成してください。以下の詳細な仕様、ディレクトリ構成、および機能要件に完全に準拠して実装してください。

### 1. プロジェクト構成とアーキテクチャ
- **フレームワーク**: `Astro` を開発サーバーとして使用。
- **アーキテクチャ**: 親となる「OS画面」のHTML内で、各アプリ（SNS、ブラウザ、チャット、写真）を `iframe` として読み込みます。
- **状態管理**: OS画面側の `public/assets/js/os-core.js` がゲーム進行度（`currentStep`）とフラグを管理します。各 `iframe` アプリは `window.parent.postMessage` を通じて親にフラグ送信やアプリ起動要求を行います。親は `activeApps` に対して状態変更を `postMessage` でブロードキャストします。

**ディレクトリ構成:**
```text
/
├── public/
│   ├── index.html          # ゲームのルートOS画面
│   ├── apps/
│   │   ├── browser.html    # ブラウザアプリ（ニュース、ラーメン店HP）
│   │   ├── chat.html       # LIMEアプリ（クイズ、ホラー展開）
│   │   ├── gallery.html    # 写真アプリ（ヒント調査）
│   │   └── sns.html        # SNSアプリ（パスコード推測用）
│   └── assets/
│       ├── css/
│       │   ├── style.css         # index.html用スタイル
│       │   └── apps-common.css   # 各アプリ共通のCSS（リセット、戻るボタン等）
│       ├── images/               # 必要な画像素材モック
│       └── js/
│           ├── os-core.js        # OSのコアロジック
│           └── app-utils.js      # postMessageラッパー等
├── src/
│   └── pages/
│       ├── index.astro     # public/index.html を読み込んで配信
│       └── hint.astro      # 攻略ページ
└── package.json            # 依存関係
```

### 2. コアシステム（デュアルデバイス・システム）
- プレイヤーは「自分のスマホ（player-phone）」と「拾ったスマホ（picked-phone-home）」の2つの画面を行き来します。
- `index.html` には、両方のコンテナを用意し、「⇄ 自分のスマホへ」などのボタンで表示を切り替える（CSSで `display` や `transform` を変更する）機能を実装してください。
- 通知（トースト）が届いた際、どちらのスマホ宛の通知かに応じて、対象のスマホコンテナをCSSアニメーションで振動（ブルブル）させてください。

### 3. ゲーム進行とストーリーフロー（State Machine）
`currentStep` は以下の順序で進行します。

1. **`intro`**: 自分のスマホで友人から「太郎のスマホを拾ったなら中を見てくれ」とチャットで言われる。
2. **`unlocked`**: 拾ったスマホのロック画面でパスコード「19940826」を入力すると解除される。
   - ※パスコードのヒントは `hint.astro` に記載（SNSのオープン日と誕生日から推測）。
3. **`news_found`**: ブラウザアプリで「東都ニュースWeb」の火葬場遺体持ち出しニュースを読む。
   - 読んだ後、自分のスマホで友人から「写真フォルダ（gallery.html）も調べてみろ」と通知が来る。
4. **`location_spec`**: 写真アプリで「厨房の写真」と「スマホケースの写真」を調査（クリックしてズーム）する。
   - 全て調査するとフラグが立ち、LIMEアプリ（chat.html）でケン（配信者）へ位置情報を送信できるようになる。
5. **`chat_sent`**: LIMEで証拠を送信すると、相手がケンではなくラーメン店主であったことが判明し、クイズ（尋問）が始まる。
6. **`clear` / `bad_end`**: クイズの結果に応じて分岐。

### 4. 各アプリの詳細仕様

#### A. 拾ったスマホのロック画面
- デジタル時計と日付を表示。
- 9桁のナンバーパッドとパスコード入力用のドット表示。
- `19940826` 入力でロック解除し、拾ったスマホのホーム画面（`#picked-phone-home`）へ移行。

#### B. SNSアプリ (`sns.html`)
- 太郎と友人の過去のSNS投稿を閲覧可能。
- ラーメン店のオープン日（2019.08.26）や、本人の誕生日（32歳の誕生日が2026.03.05）の書き込みを配置。
- 「最近だるい」「変な匂いがする」などホラーの伏線となるテキストを散りばめる。

#### C. ブラウザアプリ (`browser.html`)
- 2つのタブ（ニュース、ラーメン店公式HP）を実装。
- ニュース内の特定のテキスト「秘伝の骨」はクリック可能（CSSアニメーションで点滅させるなどアフォーダンスを持たせる）。
- これをクリックすると、ラーメン店のHPが全体的に赤黒く変貌し、文字化け（〇ンニク〇んこつ → ジンニクじんこつ）が発生するホラー演出を入れる。

#### D. 写真アプリ (`gallery.html`)
- CSS Gridで画像サムネイルを並べる。
- 「厨房の写真」をクリックし、「🔍 写真を調査する」ボタンを押すと、画像がズームされ、フラグが立つ。
- 親OSに `casePhotoViewed`, `kitchenPhotoViewed` などのフラグを送信する。

#### E. LIMEアプリ (`chat.html`)
- メッセージングアプリのUI。証拠が揃うまではロック状態。
- 証拠を送信した直後、相手が「あなた誰ですか？」と豹変し、ホラー展開へ突入。
- **3つのクイズ（尋問）**: 太郎に関する情報（誕生月、好物など）を聞かれる。
- **即バッドエンド**: 1問でも間違えると、即座にグリッチ演出と不気味なノイズが鳴り響き、バッドエンド画面に移行する。
- 3問正解でグッドエンド（警察が突入し救出）。

### 5. デザインとUX
- **スマホUI**: 拾ったスマホは、ステータスバー、時計、下部のホームバーを備えたスマートフォンライクな縦長UIにすること。PCの「デスクトップ」のような呼称・デザインは避ける。
- **カラーリング**: 全体的に暗く不気味なトーン（ダークモードベース、アクセントカラーは血を連想させる暗い赤）。
- **共通化**: 各アプリのCSSリセットやフォント、戻るボタンのデザインは `apps-common.css` に切り出すこと。

### 6. 実装の制約・ガイドライン
- 外部APIへの依存は一切なし（`fetch` も不要、全てハードコードで完結させる）。
- 最終的なクイズの「お待ちしておりますが」などの不完全な文脈はホラー演出としてそのまま実装すること。
- エラーハンドリングと状態のリセット機能（`localStorage.removeItem`）を実装すること。

上記要件を満たした上で、コードを生成し、動作するゲームを構築してください。
